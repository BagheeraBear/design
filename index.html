Interface Guidelines

Overview

What this site is

These guidelines serve as a resource to anyone developing a digital product that is connected with our applications.

It will provide you with the assets and guidance needed to create third party applications and customizations that look and feel as a part of our applications.

Who this site is for

The site should be used as a foundation by anyone using the Fortnox design and interface in a digital environment. For example agencies working with Fortnox or third party developers.

User Interface

Our user interface is carefully developed by us and used only in our applications.

When it comes to things that you create, such as third party applications, we want to encurage you to use your own interface but apply our look and feel to it where it's possible.

Things to keep in mind to create a great experience for the user is to match our colors of the buttons and input elements.

Logotype

Do's and don'ts

  • Do contact us if you have any questions regarding how to use our logotype or any other brand assets
  • Do follow the clear space guidelines described below
  • Don't modify our logotype in any way, such as changing the shape or color
  • Don't combine our logotype with your name, marks or logotype
  • Don't use any icons, images or logotypes other than what you've recieved from us to represent Fortnox

Clear space

To maximize the recognition and visibility of our brand, surround our logotype with the clear space shown in the illustration below. It should be free from any type of graphics, text or other elements that might cause confusion.

General usage

  • Use dark gray for paragraph and headline text
  • Links should use the blue color
  • The light versions of green, yellow, red and blue could all be used as status colors
  • The standard version of green, yellow, red and blue could be used as status, notification and informational colors
  • Primary action buttons should use the green color
  • Secondary action buttons should use the medium gray color
  • All buttons should use white as font color
  • Use white as the canvas background color

Typography

Arial

In our applications we use the font Arial. This is because it's a widespread font that works well on almost all devices in any size or shape.

All headings and buttons should always be uppercase while links, labels and tools should use sentence case.

Helvetica

We use the font Helvetica outside of the applications. Typical usage is this site, in advertising and in any other digital or print media.

Sentence case should be used for the most part when using Helvetica in our brand.

Icons

Using icons is a great way to explain an action or a state to the user. Try to always use labels together with icons, the meaning may be obvious to some users but not to all.

In our applications we use the icons from Font Awesome. When using these icons, try to use them in a similar way as we do in our applications.

For example: Our icon for settings () indicates some type of setting. To make it easy for the user, it would be preferable that the meaning of the icon is the same in the things you make.
